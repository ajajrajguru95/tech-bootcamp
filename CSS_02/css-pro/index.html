Introduction to CSS

CSS (Cascading Style Sheets) is what makes websites look good. It handles the colors, fonts, layouts, and overall aesthetics.

You use **HTML for the structure** and **CSS for the style**.

CSS Versions

CSS has evolved significantly over the years, from basic styling to advanced layout and animation capabilities.

  • CSS1: The foundational version, introducing basic styling for fonts, colors, and margins.
  • CSS2: Expanded upon CSS1, adding features like positioning elements, pseudo-classes, and improved layout options.
  • CSS3: This is the current and continuously evolving version. It introduces advanced features such as:
    • Animations and transitions
    • Media queries for responsive design
    • Web fonts
    • Gradients, shadows, and rounded corners
    • Flexible layout options like Flexbox and Grid Layout

How to Apply CSS (3 Ways)

There are three primary methods for applying CSS to an HTML document, each with different use cases and levels of priority.

Inline Styles
Styles written directly inside the HTML element's style attribute.
Caution: This is generally discouraged as it is messy and defeats the purpose of separation of concerns.
Internal Stylesheet
Styles written directly in a <style> tag within the HTML <head> of a single page.
Use Case: Suitable for small, single-page projects or for testing purposes.
External Stylesheet (Best Practice)
Styles are in a separate .css file, linked via the <link> tag.
This is the standard practice for large websites, promoting clean, maintainable, and reusable code.

Selectors: Who to Style

Selectors are patterns used to select the HTML elements you want to style. They form the fundamental basis of all CSS rules.

Element Selector
Targets all instances of an HTML tag (e.g., p, div).
Class Selector
Targets elements with a specific class attribute (e.g., .my-class).
ID Selector
Targets a single, unique element with a specific id attribute (e.g., #unique-id).

The "Cascade" Rule

The Cascade is the process the browser uses to decide which CSS rule applies to an element when multiple, conflicting rules target it.

  • Specificity: This determines which rule wins when multiple rules target the same element (The hierarchy is **ID > Class > Element**).
  • Source Order (Tie-breaking): If two rules have the exact same specificity, the rule declared **later** in the stylesheet wins.
  • !important: This declaration can override all other specificity rules, but its use is generally discouraged as it can lead to difficult-to-maintain code.

The CSS Box Model

The Box Model is one of the most fundamental concepts in CSS. It describes how HTML elements are structured, viewed, and allocated space on a webpage.

Every Element is a Box, composed of Content, Padding, Border, and Margin.

Basic Styling Properties: Building the Style Guide Foundation

These essential properties are the foundation of any visual design. Designers use them to define the core look and feel (the **style guide** or **design system**) before moving on to complex layouts and advanced styles.

Typography

Typography controls the appearance of text, ensuring readability and visual hierarchy.

font-family
Sets the typeface (e.g., Arial, Georgia, or a custom web font).
Linking Google Fonts
Custom fonts are typically linked in the HTML's **``** using a **``** tag, then applied with `font-family`.
font-size
Sets the size of the text (e.g., `16px`, `1.2em`, `1rem`).
font-weight
Sets the thickness of the characters (e.g., `normal`, `bold`, or numeric values like `400`, `700`).
text-align
Controls the horizontal alignment of the text (`left`, `center`, `right`, or `justify`).

Example: Setting a basic font and size for the body and linking a Google Font.

HTML (`` section):

<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap" rel="stylesheet">

CSS:


					body {
					    font-family: 'Roboto', sans-serif;
					    font-size: 16px;
					}

					h1 {
					    font-weight: 700;
					    text-align: center;
					}

Color and Backgrounds

These properties define the color palette of the interface for both foreground text and element backgrounds.

color
Sets the color of the text (foreground color). Accepts hex codes, RGB, HSL, or color names.
background-color
Sets the solid background color of the element's box.
background-image
Sets an image as the background, often used with other `background-` properties (e.g., `repeat`, `position`, `size`).

Example: Applying the primary brand color to a button.


					/* Define a variable for the primary color */
					:root {
					    --primary-color: #007bff; 
					}

					.button-primary {
					    background-color: var(--primary-color);
					    color: white;
					    border: none;
					    padding: 10px 20px;
					}

Dimensions

The **`width`** and **`height`** properties control the size of an element's content area, which is foundational for layout construction.

width
Sets the width of the content area (e.g., `100px`, `50%`, `auto`).
height
Sets the height of the content area (e.g., `50vh`, `300px`, `auto`).

Example: Setting a container to take up most of the viewport height.


					.main-container {
					    width: 90%;
					    /* 80% of the viewport height */
					    height: 80vh; 
					    margin: 0 auto; /* Center the box horizontally */
					}

Icons (Using Font Awesome)

Instead of image files, modern designs often use libraries like **Font Awesome** to include scalable, vector-based icons that can be styled easily with CSS.

How to Include Font Awesome
Link the library's CSS file in the HTML's **``** section (using a CDN or local download).
How to Use Icons
Insert an empty **`<i>`** or **`<span>`** tag with the specified classes for the desired icon (e.g., `fa-solid fa-home`).

Example: Linking the library and using a "home" icon, then styling it with CSS.

HTML (`<head>` section for linking):

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css">

HTML (`<body>` section for usage):


						<button>
						    <i class="fa-solid fa-home"></i> Home
						</button>

CSS (for styling):


						.fa-home {
						    /* Style the icon color and size */
						    color: #007bff;
						    font-size: 1.2em;
						    margin-right: 5px;
						}

Advanced CSS Properties and Concepts

Building upon the fundamentals, these advanced topics allow for complex layouts, animations, and dynamic styling necessary for modern web development.

The Display Property

The **`display`** property is fundamental to CSS layout, determining how an element behaves and interacts with other elements on the page.

block
Elements take up the full available width and start on a new line (e.g., `div`, `p`).
inline
Elements only take up as much width as necessary and flow along the same line as surrounding content (e.g., `span`, `a`).
inline-block
Elements flow inline but accept width, height, and vertical margins/padding.
none
Hides the element completely, removing it from the document flow.

CSS Positioning and Stacking

The **`position`** property is used to precisely place an element, allowing for control over the standard document flow.

static
The default position; elements are positioned according to the normal flow.
relative
Positioned relative to its normal position, affecting only itself.
absolute
Positioned relative to its nearest positioned ancestor, removed from the normal flow.
fixed
Positioned relative to the viewport, remaining in place when the page is scrolled.
sticky
Behaves as `relative` until a threshold is reached, after which it becomes `fixed`.

The **`z-index`** property controls the **stacking order** of positioned elements, determining which elements appear on top of others.

Example: Controlling the overlap of two positioned elements.


					.div-front { 
					    position: absolute; 
					    z-index: 2; /* Sits on top */
					    background-color: blue; 
					}
					.div-back  { 
					    position: absolute; 
					    z-index: 1; /* Sits underneath */
					    background-color: red; 
					}

Responsive Web Design (RWD)

RWD ensures that web pages render well on a variety of devices and screen sizes, from mobile phones to desktop monitors.

  • **Mobile-First Design:** The strategy of beginning development for mobile devices first, and then scaling up to larger screens.
  • **Media Queries:** Syntax used to apply CSS rules only when certain conditions (like screen size or orientation) are met.

Example: Using Media Queries for common breakpoints:


					/* Styles applied up to a maximum width of 768px (Mobile) */
					@media (max-width: 768px) {
					    body {
					        background-color: lightgreen;
					    }
					}

					/* Styles applied from a minimum width of 769px and up (Tablet/Desktop) */
					@media (min-width: 769px) {
					    body {
					        background-color: lightblue;
					    }
					}

CSS Variables (Custom Properties)

CSS Variables (or Custom Properties) allow you to define reusable values (like colors or fonts) that can be accessed throughout your stylesheets, simplifying maintenance and consistency.

Example: Defining and using custom properties:


					:root {
					    /* Define variables globally within the :root selector */
					    --bg-color: #333333;
					    --txt-color: white;
					}

					body {
					    /* Access variables using the var() function */
					    background-color: var(--bg-color);
					}

					h1, div {
					    color: var(--txt-color);
					}

Transitions and Transformations

These properties enable smooth, dynamic visual effects without the need for JavaScript, enhancing user experience.

  • **Transitions:** The **`transition`** property controls the speed and smoothness of property changes when an element changes state (e.g., on `hover`).
  • **Transformations:** The **`transform`** property applies 2D or 3D visual changes. Functions include:
    • `translate()` (move)
    • `rotate()` (turn)
    • `scale()` (resize)

Example: An element can be smoothly scaled up on hover using: transition: transform 0.3s ease; transform: scale(1.1);

Pseudo-classes and Pseudo-elements

These selectors target elements based on their state or position (Pseudo-classes) or allow styling of specific parts of an element (Pseudo-elements).

Pseudo-classes (Single colon: `:`)
Select elements based on a state (e.g., `:hover`, `:focus`, `:active`) or position (e.g., `:nth-child()`, `:first-child`).
Example: Styling an input field when it is active: input:focus { border-color: blue; }
Pseudo-elements (Double colon: `::`)
Select and style a specific part of an element (e.g., `::first-letter`) or insert decorative content (`::before`, `::after`).
Example: Customizing the first letter of a heading:

						h2::first-letter { 
						    font-size: 2em; 
						    text-transform: uppercase; 
						}

Attribute Selectors: You can also select elements based on their HTML attributes. Example: [type="text"] targets all text input fields.

Introduction to Layout: Flexbox and Grid

Modern CSS relies on powerful, two-dimensional layout modules—Flexbox and Grid—to arrange elements effectively and create responsive designs.

Flexbox (Flexible Box Layout)

Flexbox is a **one-dimensional** layout model primarily designed for arranging items in a single row or a single column. It's ideal for distributing space between items and aligning them.

Setting the Container
Applying **`display: flex;`** to the parent element defines it as a flex container, turning its direct children into flex items.
flex-direction
Defines the **main axis** (e.g., `row`, `row-reverse`, `column`, `column-reverse`).
justify-content
Distributes space and aligns items **along the main axis** (e.g., `center`, `flex-start`, `space-between`, `space-around`).
align-items
Aligns items **along the cross axis** (perpendicular to the main axis, e.g., `center`, `flex-start`, `stretch`).
order
A property applied to **flex items** to change their visual order, regardless of their source order in the HTML.
flex-grow / flex-shrink
Properties applied to **flex items** to determine how they absorb extra space (`grow`) or contract when space is limited (`shrink`).

Example: Centering items both horizontally and vertically using Flexbox.


					.container {
					    display: flex;
					    /* Main axis: Center items horizontally */
					    justify-content: center; 
					    /* Cross axis: Center items vertically */
					    align-items: center; 
					    height: 300px;
					}
					.item {
					    order: 3; /* Visually place this item third */
					    flex-grow: 1; /* Allow this item to take up available space */
					}

CSS Grid Layout

CSS Grid is a **two-dimensional** layout system, meaning it can handle both rows and columns simultaneously. It is the perfect tool for creating the overall page structure or complex component layouts.

Setting the Container
Applying **`display: grid;`** to the parent element defines a grid container.
grid-template-columns
Defines the number and width of the columns (e.g., `1fr 1fr 1fr`, `200px auto`).
grid-template-rows
Defines the number and height of the rows.
grid-gap (or gap)
Sets the size of the gaps (gutters) between the rows and columns.
grid-column-start/end
Properties applied to **grid items** to explicitly place them, allowing them to span multiple columns.

Example: Creating a three-column layout with consistent spacing.


					.grid-container {
					    display: grid;
					    /* Three equal-width columns (1fr = 1 fractional unit) */
					    grid-template-columns: 1fr 1fr 1fr; 
					    /* Gap between cells */
					    gap: 20px; 
					}
					.grid-item-header {
					    /* Span across all three columns */
					    grid-column: 1 / span 3; 
					}

Advanced Tools, Frameworks, and Professional Practices

Beyond writing raw CSS, professional workflows involve tools, frameworks, and techniques that enhance efficiency, maintainability, performance, and cross-browser compatibility.

CSS Preprocessors (Sass/LESS)

Preprocessors extend the capabilities of standard CSS, allowing developers to write more **maintainable and powerful** stylesheets using programming concepts.

Core Features
They introduce features like **variables**, **nesting** of selectors, reusable **mixins**, and **functions**, which are then compiled into standard CSS.
Widely Used Compilers
These tools take the preprocessor code (e.g., Sass) and convert it into browser-readable CSS.
  • **Prepros**: A cross-platform app for compiling various preprocessors.
  • **Koala**: Another popular GUI application for compiling Sass, Less, and more.

CSS Frameworks

Frameworks provide pre-written, standardized code bases and components that accelerate development by offering readily available layouts, utilities, and widgets.

  • **Tailwind CSS**: A utility-first framework where styling is applied directly in the HTML using small, single-purpose CSS classes (e.g., `flex`, `pt-4`, `text-center`).
  • **Bootstrap**: A comprehensive component library that provides pre-designed elements like navigation bars, buttons, and responsive grid systems.

Tools for Design Reference

Developers often work from digital design files created in industry-standard tools, which define the exact aesthetics and measurements for the CSS implementation.

  • **Figma**: A popular, collaborative, browser-based interface design tool.
  • **Adobe XD**: A design and prototyping tool for web and mobile apps.
  • **Webflow**: A visual development tool that bridges the gap between design and code.

Cross-Browser Adaptability

Advanced CSS knowledge includes understanding and addressing **cross-browser issues**. Since not all CSS properties are supported equally across the hundreds of browser versions, developers must verify compatibility.

The primary reference for checking which properties are supported where is the **MDN Web Docs** (Mozilla Developer Network), which provides detailed compatibility tables.

Example of a compatibility table reference:


		/* Checks compatibility for the :has() pseudo-class */
		https://developer.mozilla.org/en-US/docs/Web/CSS/:has#browser_compatibility

Performance and Development Tools

These tools are crucial for fast prototyping, debugging, and optimizing the final production code for faster loading times.

Online Code Playgrounds
Platforms like **CodePen** and **JSFiddle** allow quick prototyping and testing of small code snippets in an isolated, shareable environment.
CSS Minify
The technique of reducing the CSS file size for better performance by **clearing irrelevant characters** such as white spaces, comments, and line breaks.
Tools include preprocessors, build tools, and online services like **CSS Minifier**.

Job Roles Focused on CSS Expertise

A deep understanding of CSS is fundamental to several key roles in web development, particularly those focused on the visual layer and user experience.

UI (User Interface) Developer

The UI Developer specializes in the visual and interactive components of an application, requiring an extremely high level of CSS and preprocessor expertise.

Core Expertise
Deep knowledge in **CSS**, **Sass/Less**, and advanced concepts like **Flexbox** and **Grid** for layout.
Key Responsibilities
Building **reusable UI components** (like buttons, navigation bars, and cards).
Implementing complex **animations** and **transitions** for seamless user experiences.
Ensuring perfect **responsive design** across all devices.

Frontend Developer

The Frontend Developer acts as the bridge between the design team and the final application, ensuring the user interface is functional, interactive, and visually accurate.

Core Function
Translating **UI/UX designer wireframes** (e.g., from Figma or Adobe XD) into **functional, interactive, and pixel-perfect layouts**.
Key Skills
Strong proficiency in **HTML** for structure and **advanced CSS** for styling and layout.
Integrating styling with JavaScript frameworks (like React or Vue) to handle dynamic content.
